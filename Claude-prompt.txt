# Master Prompt ├втВмтАЭ Build a Fully-Functional Cursor-Style AI Coding Platform Prototype

> **Read this entire prompt before writing any code. This is a single, atomic instruction set. Do not skip any section.**

---

## 0. Mission

Build a **fully-functional, end-to-end interactive prototype** of a Cursor-style AI coding platform. The goal is a *clickable, live-testable* product where **every sidebar item, every dashboard widget, every button, every form, every modal, and every table is wired to working logic backed by realistic sample data**. This is **not** a static mockup, not a Figma export, and not a "coming soon" shell.

**Hard rules ├втВмтАЭ non-negotiable:**

1. **No feature is allowed to be skipped, stubbed, or marked "TODO".** If a feature is listed in section 4 below, it must be implemented and reachable from the UI.
2. **No "Coming Soon", "Not implemented", or empty placeholder pages.** Every route renders real UI with real (sample) data.
3. **Every clickable element must do something visible.** Buttons trigger state changes, open modals, update tables, push toast notifications, or navigate.
4. **Every form must validate and submit**, persist to in-memory store, and reflect changes immediately in the relevant lists, dashboards, and analytics.
5. **Every list/table must support** sort, filter, search, pagination, and row-level actions (view, edit, delete, duplicate).
6. **Every chart/metric on dashboards must read from the same sample data store** so changes propagate (e.g., creating a new Cloud Agent increments the "Active Agents" KPI).
7. **All features must be live-testable without backend setup.** Use an in-memory data layer or local storage so the reviewer can refresh and re-test.
8. **Provide seed sample data for every entity** (see section 3). Empty-state screens are allowed only when the user explicitly deletes all records.

If at any point you feel tempted to write "this would be implemented in production" ├втВмтАЭ **stop and implement it instead**. Use mocks, fakes, and timers, but make it *behave*.

---

## 1. Recommended Tech Stack

Pick one stack and stay consistent. Suggested:

- **Frontend:** Next.js 14 (App Router) + React 18 + TypeScript
- **Styling:** Tailwind CSS + shadcn/ui components + lucide-react icons
- **State:** Zustand (or Redux Toolkit) for global store; React Query for async simulation
- **Charts:** Recharts
- **Code editor:** Monaco Editor (`@monaco-editor/react`)
- **Diff viewer:** `react-diff-viewer-continued`
- **Terminal:** xterm.js
- **Markdown:** `react-markdown` + `remark-gfm`
- **Forms:** React Hook Form + Zod
- **Tables:** TanStack Table
- **Mock backend:** MSW (Mock Service Worker) OR a single `lib/mockApi.ts` returning Promises with `setTimeout` delays (300├втВмтАЬ800ms) to simulate latency
- **Persistence:** `zustand/persist` to localStorage so refreshes preserve state

If you choose a different stack (SvelteKit, Vue/Nuxt, etc.) the **rules and feature list still apply 1:1**.
